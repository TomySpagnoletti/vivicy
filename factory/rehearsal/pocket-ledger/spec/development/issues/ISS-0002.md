# ISS-0002 - Read expenses from the Ledger as defensive copies

## Summary

Implement the Ledger's retrieval reads: list all expenses in insertion order and read a single expense by id. An unknown id returns no record rather than throwing, and returned values are copies so a caller can never mutate the stored record.

## Scope

Add two read operations to the Ledger built in ISS-0001:

- a read of all expenses returned in insertion order;
- a read of a single expense by id.

Reading an unknown id returns no record (for example `undefined`) rather than throwing. The returned collections and records are copies: mutating a returned value must never alter the stored record.

## Out Of Scope

Adding or validating expenses (ISS-0001), totals (ISS-0003), and all downstream consumers (categorizer, reporter, exporter, CLI). No new validation rules are introduced here.

## Verification

Unit tests in `test/` (node:test) to full branch coverage: listing returns all stored expenses in insertion order; reading a known id returns that record; reading an unknown id returns no record without throwing; mutating a value returned from the list read does not alter the stored record; mutating a value returned from the by-id read does not alter the stored record. Tests assert real returned state and real isolation of the store. The deterministic gate `gate:test:ledger-retrieval` must be green (node --test exit 0) before this issue is reported complete.
