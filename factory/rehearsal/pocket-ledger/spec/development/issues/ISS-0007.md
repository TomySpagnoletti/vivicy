# ISS-0007 - Serialize the Ledger to deterministic CSV

## Summary

Implement the Exporter: a pure read of the Ledger that serializes expenses to a deterministic CSV string with a fixed header and column order, rows in insertion order, RFC-style quoting of special characters, line-feed row separators, and plain-integer `amountCents`. An empty Ledger exports just the header row.

## Scope

Build the Exporter as a pure-JS function under `src/` that reads expenses from the Ledger (via the ISS-0002 reads) and returns a deterministic CSV string. The export is a pure read and must not mutate the Ledger.

Column order: a fixed header row with columns in exactly this order: `id,date,description,category,amountCents`. Every data row follows the same column order.

Row order: data rows are emitted in Ledger insertion order, matching the order in which expenses were added.

Quoting rules: a field that contains a comma, a double quote, or a newline must be wrapped in double quotes; a double quote inside a quoted field is escaped by doubling it; fields without those characters are emitted unquoted.

Line endings and amount format: rows are separated by a single line feed (`\n`); `amountCents` is emitted as a plain integer with no currency symbol, thousands separator, or decimal point. An empty Ledger exports just the header row.

## Out Of Scope

Computing totals (ISS-0003), reporting (ISS-0005, ISS-0006), and CLI wiring (ISS-0008). This issue does not write to a file or to standard output; it returns the CSV string.

## Verification

Unit tests in `test/` (node:test) to full branch coverage: an empty Ledger exports exactly the header row `id,date,description,category,amountCents`; a single expense produces a header plus one data row in the fixed column order; multiple expenses appear as rows in insertion order; a field containing a comma is wrapped in double quotes; a field containing a double quote is wrapped and the inner quote is doubled; a field containing a newline is wrapped in double quotes; fields without special characters are emitted unquoted; rows are joined with a single `\n`; `amountCents` is emitted as a bare integer. Tests assert the exact emitted string. The deterministic gate `gate:test:csv-export` must be green (node --test exit 0) before this issue is reported complete.
