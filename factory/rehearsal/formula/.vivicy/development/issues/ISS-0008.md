# ISS-0008 - Logical Functions

## Summary

This issue registers the logical function family on the Registry. It implements the shared truthiness rule (a boolean is itself, a number is false when zero and true otherwise, an empty cell value is false, an error value propagates unchanged, and any other type yields `#VALUE!`), IF with error-condition propagation and a defaulted false branch, AND and OR over flattened ranges requiring at least one non-empty argument, and single-argument NOT. Logical functions return a boolean except IF, which returns one of its branch values.

## Scope

Register the logical functions on the Registry using the shared calling convention from ISS-0006. Implement the truthiness rule shared across the family: a logical function treats a boolean as itself, a number as false when it is zero and true otherwise, and an empty cell value as false; a value being truthiness-tested that is an error value propagates unchanged (per the calling convention), and any other value that is neither boolean, number, nor empty yields a `#VALUE!` error value. Logical functions return a boolean except `IF`, which returns one of its branch values.

Implement `IF(condition, whenTrue, whenFalse)`: it evaluates the condition to a boolean and returns `whenTrue` when it is true and `whenFalse` when it is false. When the condition is an error value, `IF` returns that error value (the error-aware override), and a missing third argument defaults its false branch to the boolean false.

Implement `AND` and `OR`: `AND` returns true only when every argument is truthy and false otherwise, while `OR` returns true when at least one argument is truthy and false otherwise. Both accept ranges, flatten them in range order, skip empty cell values, and require at least one non-empty argument. Implement `NOT(value)`: it returns the boolean negation of its single argument's truthiness — true becomes false and false becomes true — requires exactly one argument, and yields a `#VALUE!` error value when given the wrong number of arguments.

## Out Of Scope

The Registry, calling convention, default error propagation, coercion, and range flattening (owned by ISS-0006, consumed here). The math, text, and lookup families (their own issues).
