# ISS-0010 - Lookup Function VLOOKUP

## Summary

This issue registers VLOOKUP on the Registry. It searches the first column of a two-dimensional range for the key and returns the same-row value at a one-based columnIndex, supporting an exact-match mode and a sorted-ascending approximate-match mode. It returns `#REF!` for an out-of-bounds columnIndex, `#VALUE!` for a key not found in exact-match mode, and `#VALUE!` for an empty range or a non-range value supplied as the range argument; an error value passed as the key propagates unchanged.

## Scope

Register `VLOOKUP(key, range, columnIndex, exactMatch)` on the Registry using the shared calling convention from ISS-0006. It searches the first column of the given range for the key and returns the value in the same row at the one-based `columnIndex`. The range argument is a two-dimensional range of cell values — the rectangular block the calling convention delivers, which a lookup function reads by its rows and columns — and `columnIndex` counts columns from one starting at the range's first column.

Implement both match modes. When `exactMatch` is true, `VLOOKUP` returns the value from the first row whose first-column value equals the key. When `exactMatch` is false, the first column is assumed sorted ascending and `VLOOKUP` returns the value for the largest first-column entry that is less than or equal to the key.

Implement the bounds and error cases. A `columnIndex` below one or greater than the range's column count yields a `#REF!` error value, and a key not found in exact-match mode yields a `#VALUE!` error value. An empty range, or a non-range value supplied as the `range` argument, yields a `#VALUE!` error value. Any error value passed as the key propagates unchanged (the error-aware override).
