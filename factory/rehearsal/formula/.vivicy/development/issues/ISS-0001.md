# ISS-0001 - Error Values and Propagation Contract

## Summary

This issue establishes the engine's typed error model. It defines exactly four spreadsheet error values — `#DIV/0!`, `#VALUE!`, `#NAME?`, and `#REF!` — as distinct, comparable sentinels that can never collide with ordinary cell text, exposes a predicate that reports whether a value is an error and a reader that returns an error's code string, and specifies the default first-error-wins propagation contract that every operator and function later relies on. It is the foundation every other module imports for raising and recognizing errors.

## Scope

Define the closed set of exactly four error values with their canonical meanings: `#DIV/0!` for division by zero, `#VALUE!` for a wrong-type or uncoercible operand, a malformed or unparseable formula, or an operation with no valid result value, `#NAME?` for an unknown function name or unlexable input, and `#REF!` for an invalid or cyclic reference. Each error value is a distinct, comparable sentinel — never a plain string — so it can never collide with or be mistaken for cell text. The exact four code strings are a fixed, closed set defined once in this module and reused everywhere; no other module invents a new code or reformats an existing one.

Expose two inspection primitives: a predicate that returns whether a given value is one of the error sentinels, and a reader that returns the error's code string. Error values are never silently coerced to numbers, strings, or booleans by these primitives.

Provide the default propagation helper that the operators and functions consume: given an ordered list of operands or arguments, it returns the first error value in left-to-right evaluation order, or signals that none is present. This encodes the rule that an error operand of an operator, or an error argument of a function, yields that same error value, with the first in evaluation order winning. The contract permits later functions to override this default (error-aware and error-skipping functions), but this module owns the default and the sentinels they build on. Producing an error value is the engine's only spreadsheet-level failure channel: error production never throws.

## Out Of Scope

How individual operators, function families, the Evaluator, or recalculation actually invoke propagation — those issues consume this contract. The error-aware overrides (`IF`, `VLOOKUP`) and error-skipping override (`COUNT`) are specified and implemented in their own function issues; this issue only defines the default they deviate from. Tokenizer/parser error production is wired in their issues using these sentinels.
