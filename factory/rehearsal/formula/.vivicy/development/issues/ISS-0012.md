# ISS-0012 - Dependency Graph and Recalculation

## Summary

This issue implements the DepGraph and the recalculation it drives. The DepGraph records, for each cell that holds a formula, the set of cells its formula reads, exposes the direct dependents of a cell, and computes a recalculation order, detecting and reporting a dependency cycle. Recalculation recomputes only the cells that transitively depend on a changed cell in topological dependency order, resolves cells participating in a cycle to `#REF!` while still terminating for every other cell, and produces the same values regardless of the order cells were set.

## Scope

Implement the DepGraph: it records, for each cell that holds a formula, the set of cells its formula reads. It exposes the direct dependents of a cell and computes a recalculation order, and a dependency cycle is detected and reported to recalculation.

Implement recalculation against the Sheet (ISS-0003) and Evaluator (ISS-0011). Recalculation keeps formula cells consistent with the cells they depend on: when a cell's stored value changes, every cell that transitively depends on it is recomputed in dependency order. The recalculation order is a topological sort of the dependency graph — a cell is recomputed only after every cell it reads has been recomputed — and cells that do not depend, directly or transitively, on a changed cell are not recomputed.

Implement cycle handling and determinism. If the dependency graph contains a cycle, the cells that participate in the cycle each resolve to a `#REF!` error value, and recalculation still terminates for every cell outside the cycle; a cycle never causes infinite recomputation. Given the same stored cells and the same formulas, recalculation produces the same values every time, independent of the order in which cells were set, reads stored state through the Sheet, and never mutates a cell the formula did not target.
