# ISS-0003 - Sheet Cell Storage and Reads

## Summary

This issue implements the Sheet, the single owner of stored cell values. It stores a value per cell keyed by its normalized A1 address, returns an empty cell value when an address that was never set is read, reads a single cell value by address, and reads a range as a rectangular block of rows and columns. Returned range blocks are defensive copies so a caller mutating a returned block can never alter stored cell values, and reading is a pure query that never changes stored state.

## Scope

Implement cell storage: the Sheet stores a value per cell address keyed by its normalized A1 address, using CellRef for address normalization. Setting a cell records its value, and the Sheet is the only owner of stored cell values — no other module mutates a stored value directly. Reading an address that was never set returns an empty cell value rather than throwing.

Implement reads: the Sheet reads a single cell value by address, and reads a range of cell values as a rectangular block of its rows and columns (an ordered structure of rows, each an ordered list of column values), using CellRef to expand the range. Returned range blocks are copies: mutating a returned block never alters stored cell values. Reading is a pure query and never changes stored state.

## Out Of Scope

A1 address and range parsing/expansion (owned by the CellRef issue, consumed here). Formula storage, dependency registration, and recalculation (owned by the DepGraph and recalculation issue). Reference resolution during evaluation (owned by the Evaluator issue). This issue stores and returns raw cell values; it does not interpret formulas.
