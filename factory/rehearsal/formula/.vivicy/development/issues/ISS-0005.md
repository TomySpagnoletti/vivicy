# ISS-0005 - Pratt Parser and AST

## Summary

This issue implements the Parser, a precedence-climbing (Pratt) parser that consumes the Tokenizer's token list and produces an abstract syntax tree. It builds number and string literal nodes, ref nodes, unary nodes, binary nodes, and call nodes, encoding operator precedence and associativity and parenthesized grouping. A `name` token immediately followed by a left parenthesis begins a function call with comma-separated argument expressions and an optional empty argument list, and the function name is preserved verbatim. A token sequence that does not form a complete expression, an unmatched parenthesis, or a trailing token reports a `#VALUE!` error result rather than throwing.

## Scope

Implement a precedence-climbing (Pratt) parser that consumes the token list and produces an AST. It produces these node kinds: `number` and `string` literal nodes, a `ref` node carrying a cell reference or range, a `unary` node for a prefix operator, a `binary` node for an infix operator with a left and right child, and a `call` node carrying a function name and an ordered list of argument nodes.

Encode precedence and associativity: comparison operators bind most loosely, then addition and subtraction, then multiplication and division, then exponentiation; the arithmetic operators are left-associative except exponentiation, which is right-associative. A unary minus binds tighter than any binary operator. Parentheses override precedence by grouping a sub-expression.

Handle function calls: a `name` token immediately followed by a left parenthesis begins a function call; arguments are expressions separated by commas, and an empty argument list is allowed. The function name is preserved verbatim so the Evaluator can look it up in the Registry. A token sequence that does not form a complete expression, an unmatched parenthesis, or a trailing token after a complete expression reports a `#VALUE!` error result (using the error sentinel from ISS-0001) rather than throwing.

## Out Of Scope

Lexing the input into tokens (owned by the Tokenizer issue, consumed here). Evaluating the AST, resolving function names against the Registry, or resolving references against the Sheet (owned by the Evaluator issue). This issue produces structure; it does not compute values.
