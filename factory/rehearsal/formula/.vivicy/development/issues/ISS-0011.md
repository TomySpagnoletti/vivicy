# ISS-0011 - Evaluator Tree Walk

## Summary

This issue implements the Evaluator that walks an abstract syntax tree and reduces it to a single value as a pure function of the tree and the Sheet, storing no state between calls. It evaluates literal, binary, unary, call, and ref nodes: arithmetic and comparison operators including division-by-zero, first-error operator propagation with non-short-circuiting call nodes, case-insensitive Registry dispatch with `#NAME?` for an unknown function, and single-cell and range reference resolution against the Sheet with `#REF!` for a structurally invalid reference.

## Scope

Implement the Evaluator: it walks an abstract syntax tree (from ISS-0005) and reduces it to a single value. It is a pure function of the tree and the Sheet (ISS-0003) it is given and stores no state between calls.

Implement literals and operators. A `number` or `string` literal node evaluates to its literal value. A `binary` arithmetic node applies the operator to its evaluated children using JavaScript number arithmetic; a comparison node yields a boolean. A `unary` minus negates its evaluated child. Division whose right operand evaluates to zero produces a `#DIV/0!` error value.

Implement error propagation. If any operand of an operator node evaluates to an error value, the node evaluates to that same error value without applying the operator, and the first error in evaluation order is the one propagated (using the first-error helper from ISS-0001). A `call` node does not short-circuit this way: it passes its evaluated arguments, error values included, to the function, which handles errors per its own contract. An arithmetic operator applied to a non-numeric, non-coercible operand produces a `#VALUE!` error value.

Implement function dispatch and reference resolution. A `call` node resolves its function name against the Registry (ISS-0006) case-insensitively, evaluates the argument nodes left to right, and invokes the function with the evaluated arguments, error values included; a name that is not registered makes the node evaluate to a `#NAME?` error value. A `ref` node for a single cell resolves to that cell's current value through the Sheet; a `ref` node for a range resolves to the rectangular block of cell values the range covers — its rows and columns; a reference whose address does not exist in the Sheet contributes an empty value, and a structurally invalid reference produces a `#REF!` error value.

## Out Of Scope
