# ISS-0002 - CellRef A1 Addresses and Ranges

## Summary

This issue implements the CellRef module that understands A1 notation. It parses a cell address such as `A1` or `AB12` into zero-based column and row indices and back, normalizing case-insensitive column letters to upper case and rejecting syntactically invalid addresses. It also parses a colon range such as `A1:B3`, expands it into the ordered list of addresses it covers iterating row by row, and normalizes a reversed range so the smaller column and row come first.

## Scope

Implement A1 cell-address parsing: an address is one or more column letters followed by a one-based row number. Column letters are case-insensitive and normalized to upper case. The module converts an address to a zero-based column index and a zero-based row index, and converts a zero-based column/row index pair back to its normalized A1 string. A syntactically invalid address (missing column, missing row, zero or negative row, non-A1 characters) is rejected.

Implement range handling: a range is two cell addresses separated by a colon, such as `A1:B3`. The module expands a range into the ordered list of the addresses it covers, iterating row by row from the top-left corner to the bottom-right corner. A reversed range (where the second corner is above or to the left of the first) is normalized so the smaller column index and smaller row index form the top-left corner before expansion.

## Out Of Scope

Storing or reading cell values (owned by the Sheet issue), resolving references during evaluation (owned by the Evaluator issue), and the rectangular-block read shape (owned by the Sheet issue). This issue covers address/range syntax and index conversion only, not value lookup.
