# ISS-0004 - Categorize an expense by ordered keyword rules

## Summary

Implement the Categorizer: a pure function that assigns a category label to an expense from a deterministic, ordered list of keyword rules applied to the expense description. The first matching rule wins, matching is case-insensitive, and when no rule matches the fixed `uncategorized` label is assigned.

## Scope

Build the Categorizer as a pure-JS function under `src/` that takes an expense description and an ordered list of category rules and returns a category label.

A category rule has a `category` label and a list of lowercase `keywords`. A rule matches when the lowercased description contains at least one of the rule's keywords as a substring. Rules are evaluated in declaration order; the first matching rule wins and later rules are not consulted once a match is found. Evaluation is case-insensitive.

When no rule matches, the Categorizer assigns the category `uncategorized`. That default label is a fixed constant and is never configurable per call.

Categorization is a pure function of the description and the rule list: the same description and rule list always produce the same category, and the Categorizer holds no mutable state between calls.

## Out Of Scope

Storing expenses (ISS-0001), reading the Ledger, and CLI wiring (ISS-0008, which calls this function when adding an expense). This issue does not persist the assigned category; it returns it.

## Verification

Unit tests in `test/` (node:test) to full branch coverage: a description that matches the first rule returns that rule's category; a description that matches only a later rule returns the later rule's category; a description matching multiple rules returns the first matching rule's category (order precedence); matching is case-insensitive across both description casing and keyword presence; a description matching no rule returns `uncategorized`; an empty rule list returns `uncategorized`; calling twice with identical inputs returns identical results (purity).
